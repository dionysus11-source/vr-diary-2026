# Capability: VR Investment Diary (Web App)

Value Rebalancing(VR) 투자 진행 상황을 추적하고 대시보드에서 시각화하는 시스템입니다.

## ADDED Requirements

### Requirement: 웹 기반 데이터 입력 UI
시스템은 사용자가 웹 브라우저를 통해 V, 현재 평가금, 현재 현금 풀 데이터를 입력할 수 있는 폼(Form)을 SHALL 제공해야 합니다.

#### 시나리오: 사용자가 웹 화면에서 데이터를 입력함
- **WHEN** 사용자가 웹 대시보드에서 3가지 필드를 입력하고 저장 버튼을 누르면
- **THEN** 데이터가 백엔드로 전송되어 CSV에 저장되고 화면이 갱신됩니다.

### Requirement: 인터랙티브 그래프 시각화
시스템은 투자 데이터를 시각화하기 위한 고품질의 인터랙티브 그래프를 SHALL 생성해야 합니다.
- 그래프에는 다음 항목이 표시되어야 합니다: V, V max, V min, **총 자산(평가금+Pool)**, **B&H 총 자산(v_bh+초기 Pool)**.
- 특정 지점에 마우스를 올리면 해당 날짜의 상세 수치가 팝업으로 표시되어야 합니다.

#### 시나리오: 사용자가 그래프의 상세 수치를 확인함
- **WHEN** 사용자가 특정 날짜의 그래프 포인트 위에 마우스를 올리면
- **THEN** 해당 시점의 V, 평가금, 풀 잔액 등의 정보가 툴팁으로 나타납니다.

### Requirement: 데이터 영속성 (CSV)
시스템은 웹 앱 종료 후에도 데이터가 유지되도록 백엔드에서 CSV 파일에 투자 이력을 SHALL 저장해야 합니다.

#### 시나리오: 서버 재시작 후 데이터 로드
- **WHEN** 웹 앱을 새로고침하거나 서버를 재시작하면
- **THEN** CSV 파일에 저장된 모든 과거 이력이 차트에 다시 표시됩니다.

### Requirement: Buy & Hold 실시간 비교 (총 자산 기준)
시스템은 전략의 성과를 공정하게 비교하기 위해 **총 자산(평가금+Pool)**과 **B&H 총 자산(v_bh+초기 Pool)**을 SHALL 실시간으로 계산하여 그래프에 함께 표시해야 합니다.

#### 시나리오: 총 자산 기반 비교 라인 표시
- **WHEN** 대시보드가 로드될 때
- **THEN** 현재의 총 자산 추이와 가만히 있었을 때의 B&H 총 자산 추이가 차트에 그려집니다.
