## ADDED Requirements
### Requirement: 무한매수법 버전 선택
시스템 SHALL 신규 회차 생성 시 무한매수법 버전을 선택할 수 있게 해야 한다(MUST).

#### Scenario: 회차 생성 시 버전 선택
- **WHEN** 사용자가 새로운 회차를 생성한다
- **THEN** 무한매수법 2.2 버전과 4.0 버전 중 하나를 선택할 수 있어야 한다
- **AND** 버전 선택 필드의 기본값은 2.2 버전으로 설정된다
- **AND** 선택된 버전 정보는 회차 데이터의 일부로 서버에 저장된다

### Requirement: 무한매수법 V4.0 일반/리버스모드 가이드 제공
시스템 SHALL 회차의 버전이 4.0인 경우, 일반모드와 리버스모드 상태를 추적하여 해당 룰에 맞는 맞춤형 매수/매도 가이드를 제공해야 한다(MUST).

#### Scenario: V4.0 일반모드 가이드 표시
- **WHEN** 사용자가 버전 4.0 (일반모드)으로 설정된 회차의 상세 모달을 연다
- **THEN** 무한매수 4.0 일반모드 룰(`doc/무한매수_일반모드.md` 참조)에 따른 매수 및 매도 가이드 섹션을 명확하게 표시한다
- **AND** T값 수준(전반전/후반전)에 따른 매수 방법(평단/별지점 LOC 분할 등)을 다르게 안내한다
- **AND** 가변적인 별지점(★)을 계산하여 화면에 표시한다

#### Scenario: V4.0 리버스모드 전환 및 가이드 표시
- **WHEN** 버전 4.0 일반모드 진행 중 T값이 전체 분할수를 초과하여 원금이 소진된다
- **THEN** 회차 상태를 '리버스모드'로 자동 전환한다
- **AND** V4.0 리버스모드 룰(`doc/무한매수_리버스모드.md` 참조)에 따라 첫날 무조건 매도(MOC), 둘째 날 이후 별지점 LOC 매도/매수 안내를 표시한다
- **AND** 직전 5거래일 종가 평균 기반의 새로운 별지점(★)을 계산하여 표시한다

#### Scenario: V4.0 리버스모드 종료 및 복귀
- **WHEN** 리버스모드 중 종가가 평단 대비 특정 기준(TQQQ -15%, SOXL -20%) 이상으로 회복된다
- **THEN** 즉시 리버스모드를 종료하고 일반모드로 복귀한다
- **AND** 리버스모드에서 사용하던 T값을 그대로 일반모드로 넘겨 사용하도록 가이드를 업데이트한다
