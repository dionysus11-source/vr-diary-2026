# 변경: 라오어 밸류 리밸런싱 CLI 프로그램 추가

## 왜 (Why)
레버리지 ETF(TQQQ) 투자 시 밸류 리밸런싱 전략을 체계적으로 관리할 수 있는 CLI 도구가 필요합니다. 수동 계산의 오류를 줄이고, 2주 단위 리밸런싱과 5주 분할 매수 시뮬레이션을 자동화하여 투자 의사결정을 지원합니다.

## 무엇을 변경하는가 (What Changes)

### 새로운 기능
- **밸류 계산**: 2주마다 금요일 TQQQ 종가를 기반으로 V2 계산
  - 공식: `V2 = v1 + (pool/10) + ((E-v1)/(2×√10))`
  - E = 금요일 종가 × 주식수 (평가금액)
  - v1 = 이전 차수의 V값
  - pool = 현금
  - G = 10 (고정)

- **자동 가격 데이터**: 야후 파이낸스 API 등에서 금요일 TQQQ 종가 자동 가져오기

- **리밸런싱 신호**: V2 값의 ±15% 범위를 벗어나면 매수/매도 신호 제공

- **5주 분할 매수 시뮬레이션**: 5주 단위로 분할 매수 시 각 주차별 매수/매도 시점과 예상 밸류 변화 표

- **데이터 관리**: 기존 기록 조회, 편집, 삭제 기능

### 기술적 요소
- Python CLI 프로그램 (vr_record 디렉토리 내)
- CSV 파일로 데이터 영구 저장
- 색상 지원 콘솔 UI (Windows 10+ 및 Unix)
- 야후 파이낸스 API 연동 (yfinance 라이브러리)

## 영향 (Impact)

### 관련 스펙
- 새로운 스펙 생성: `value-rebalancing-cli`

### 관련 코드
- 새로운 디렉토리: `vr_record/value_rebalancing/`
  - `main.py` - CLI 진입점
  - `calculator.py` - 밸류 계산 로직
  - `data_manager.py` - CSV 데이터 관리
  - `price_fetcher.py` - API 가격 조회
  - `simulator.py` - 5주 분할 매수 시뮬레이션

### 사용자 시나리오
1. 사용자가 초기 자본(현금, 주식수)을 설정
2. 2주마다 금요일 자동으로 V값 계산 및 리밸런싱 신호 확인
3. 5주 분할 매수 계획 시뮬레이션으로 매수/매도 시점 파악
4. 역사적 기록 조회 및 편집

### 의존성
- Python 3.8+
- `yfinance` - TQQQ 가격 데이터
- `colorama` - 크로스 플랫폼 색상 출력
- `pandas` - 데이터 처리 (선택)

## 비기능 요구사항
- **데이터 프라이버시**: 모든 데이터 로컬 CSV 저장, 외부 전송 없음
- **오프라인 지원**: 마지막으로 가져온 가격 데이터로 오프라인 계산 가능
- **간결한 UI**: exchange-rate-tracker와 유사한 메뉴 기반 CLI
