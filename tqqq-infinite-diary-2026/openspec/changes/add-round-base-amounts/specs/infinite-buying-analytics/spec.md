## MODIFIED Requirements
### Requirement: 매수/매도 기록 입력
시스템 SHALL 순차적인 매수/매도 기록 입력을 지원해야 한다(MUST).

#### Scenario: 순차적 입력 흐름
- **WHEN** 사용자가 새로운 거래를 입력한다
- **THEN** 다음 순서대로 입력 필드를 제공한다:
  1. 종목 선택 (TQQQ, SOXL 등 - 추후 추가 가능)
  2. 총 투자 원금 (Seed) 입력
  3. 1회 매수 시도액 입력
  4. 거래 유형 선택 (매수 또는 매도)
  5. 가격 입력
  6. 수량 입력
  7. 날짜 입력 (엔터치면 오늘 날짜 자동 입력, 한국시간 기준)
- **AND** 최초 회차 생성 시 총 투자 원금과 1회 매수 시도액을 명시적으로 입력받는다

### Requirement: 회차 상세 모달
시스템 SHALL 카드 클릭 시 회차 상세를 모달로 표시해야 한다(MUST).

#### Scenario: 모달 내용 - 기본 정보 및 매수/매도 가이드
- **WHEN** 회차 상세 모달이 열린다
- **THEN** 기존 기본 정보(회차 번호, 종목명 등)와 함께, **무한매수법 V2.2 매수/매도 가이드 섹션**을 시각적으로 가장 잘 보이게 표시한다
- **AND** 상단에 현재 상태(전반전, 후반전, 쿼터손절) 뱃지와 함께 T값 및 Star값 카드를 배치한다
- **AND** 쿼터 손절모드에서 매도나 매수가 성립되어 후반전 복귀 조건을 충족하면, 즉각적으로 상태 뱃지가 '후반전'으로 다시 업데이트되어 현재 상태를 명확히 알 수 있게 표시한다
- **AND** 기본 정보 내역에 해당 회차의 '총 투자 원금(Seed)' 및 '1회 매수 시도액'을 포함하여 표시한다

#### Scenario: 모달에서 편집 기능
- **WHEN** 사용자가 모달 하단의 편집 버튼을 클릭한다
- **THEN** 다음 기능을 제공한다:
  - **매수 추가**: 진행중 회차에 추가 매수 입력
  - **매도 추가**: 보유 수량이 있을 때 매도 입력
  - **거래 수정**: 개별 매수/매도 내역 수정 (가격, 수량, 날짜)
  - **거래 삭제**: 개별 매수/매도 내역 삭제
  - **회차 설정 편집**: 회차의 총 투자 원금 및 1회 매수 시도액 정보를 명시적으로 수정
  - **회차 삭제**: 회차 전체 삭제 (확인 절차 포함)
  - **가격 새로고침**: 진행중 회차의 최신 종가 다시 가져오기

## ADDED Requirements
### Requirement: V2.2 계산 기준 설정
시스템 SHALL V2.2 가이드의 T값 계산 시 명시된 1회 매수 시도액을 우선적으로 적용해야 한다(MUST).

#### Scenario: 명시된 1회 매수 시도액 활용
- **WHEN** 회차 내에 사용자가 명시적으로 입력한 '1회 매수 시도액' 데이터가 존재할 때
- **THEN** 시스템은 V2.2의 T값 계산 시 첫 주문 내역의 합계가 아닌 해당 지정 변수를 기준값으로 사용하여 정확한 T값을 산출한다
- **AND** 설정값이 없는 구버전 기록에 대해서만 호환성을 위해 기존 '첫 매수액'을 fallback으로 활용한다
