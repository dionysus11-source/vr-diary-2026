## ADDED Requirements

### Requirement: Exchange Rate Recording
시스템 SHALL 사용자가 환전 날짜, 환율, 금액을 기록할 수 있어야 한다.

#### Scenario: 환율 기록 성공
- **WHEN** 사용자가 환전 날짜, 환율(원/달러), 환전 금액(달러)을 입력한다
- **THEN** 시스템은 해당 정보를 CSV 파일에 저장한다
- **AND** 저장 성공 메시지를 표시한다

#### Scenario: 환율 기록 실패 - 잘못된 입력
- **WHEN** 사용자가 음수의 환율이나 금액을 입력한다
- **THEN** 시스템은 오류 메시지를 표시한다
- **AND** 데이터를 저장하지 않는다

### Requirement: Exchange Statistics Display
시스템 SHALL 현재까지의 환전 통계 정보를 표시할 수 있어야 한다.

#### Scenario: 통계 표시 성공
- **WHEN** 사용자가 통계 조회를 선택한다
- **THEN** 시스템은 총 환전 횟수를 표시한다
- **AND** 총 환전 달러 금액을 표시한다
- **AND** 평균 환율(가중 평균)을 표시한다
- **AND** 총 투자 원화 금액을 표시한다

#### Scenario: 통계 표시 - 데이터 없음
- **WHEN** 사용자가 통계 조회를 선택한다
- **AND** 저장된 환전 기록이 없다
- **THEN** 시스템은 "기록된 데이터가 없습니다" 메시지를 표시한다

### Requirement: Re-exchange Profit Calculator
시스템 SHALL 특정 환율에서 재환전 시 이익/손실을 계산할 수 있어야 한다.

#### Scenario: 이익 계산 성공
- **WHEN** 사용자가 목표 환율을 입력한다
- **THEN** 시스템은 목표 환율로 재환전 시 수익률을 계산한다
- **AND** 예상 수익(원화)을 표시한다
- **AND** 수익률(백분율)을 표시한다
- **AND** 이익/손실 여부를 표시한다

#### Scenario: 이익 계산 - 데이터 없음
- **WHEN** 사용자가 목표 환율을 입력한다
- **AND** 저장된 환전 기록이 없다
- **THEN** 시스템은 "먼저 환전 기록을 추가해주세요" 메시지를 표시한다

### Requirement: Console User Interface
시스템 SHALL 사용자 친화적인 콘솔 인터페이스를 제공해야 한다.

#### Scenario: 메인 메뉴 표시
- **WHEN** 프로그램이 시작된다
- **THEN** 시스템은 다음 옵션들을 표시한다:
  - 1. 환율 기록 추가
  - 2. 전체 기록 보기
  - 3. 통계 보기
  - 4. 재환전 이익 계산
  - 5. 종료

#### Scenario: 메뉴 선택 및 실행
- **WHEN** 사용자가 메뉴 번호를 입력한다
- **THEN** 시스템은 해당 기능을 실행한다
- **AND** 실행 완료 후 메인 메뉴로 돌아간다

#### Scenario: 잘못된 메뉴 선택
- **WHEN** 사용자가 유효하지 않은 메뉴 번호를 입력한다
- **THEN** 시스템은 "잘못된 선택입니다" 메시지를 표시한다
- **AND** 메인 메뉴를 다시 표시한다

### Requirement: Data Persistence
시스템 SHALL 모든 환전 기록을 CSV 파일에 영구 저장해야 한다.

#### Scenario: 데이터 자동 저장
- **WHEN** 사용자가 환전 기록을 추가한다
- **THEN** 시스템은 즉시 CSV 파일에 데이터를 추가한다
- **AND** 파일이 없는 경우 새로 생성한다

#### Scenario: 데이터 로드
- **WHEN** 프로그램이 시작된다
- **THEN** 시스템은 기존 CSV 파일에서 데이터를 로드한다
- **AND** 파일이 없는 경우 빈 데이터로 시작한다
