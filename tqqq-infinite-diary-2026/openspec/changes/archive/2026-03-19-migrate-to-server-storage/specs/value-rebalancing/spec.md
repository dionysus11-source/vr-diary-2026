## ADDED Requirements
### Requirement: 서버 기반 밸류리밸런싱 데이터 영속성 보존
시스템 SHALL 밸류리밸런싱 일지 및 투자 기록을 기존 브라우저 저장소가 아닌 백엔드 서버(DB)에 영구적으로 보존해야 한다(MUST).

#### Scenario: 교차 기기 및 브라우저 환경 동기화
- **WHEN** 사용자가 데스크톱에서 작업을 마친 후 모바일 등 다른 기기나 브라우저에서 밸류리밸런싱 메뉴에 접속한다
- **THEN** 서버로부터 최신 데이터를 불러와 동일한 밸류리밸런싱 투자 현황을 제공한다
- **AND** 브라우저 로컬 캐시는 오프라인 상태 보조용으로만 제한적으로 활용하며 서버 데이터베이스를 최우선 진실의 원천(Single Source of Truth)으로 사용한다

#### Scenario: 데이터 저장 및 API 통신 처리
- **WHEN** 사용자가 밸류리밸런싱 거래 기록을 추가 혹은 수정, 삭제한다
- **THEN** 즉각적으로 백엔드 서버 API를 호출하여 데이터베이스에 변경 상태를 반영한다
- **AND** 네트워크 에러나 응답 지연 발생 시 UI에 명확한 진행/실패 상태를 피드백(toast 알림 등)으로 제공한다
