## ADDED Requirements

### Requirement: Dollar Usage Recording
시스템 SHALL 사용자가 보유 달러를 사용한 기록을 추가할 수 있어야 한다.

#### Scenario: 달러 사용 기록 성공
- **WHEN** 사용자가 사용 날짜, 금액(달러), 용도, 메모를 입력한다
- **THEN** 시스템은 해당 정보를 CSV 파일에 저장한다 (type: 'usage')
- **AND** 저장 성공 메시지를 표시한다

#### Scenario: 달러 사용 기록 실패 - 잔액 부족
- **WHEN** 사용자가 현재 보유 달러보다 큰 금액을 사용하려고 한다
- **THEN** 시스템은 "잔액이 부족합니다" 메시지를 표시한다
- **AND** 현재 보유 달러와 입력한 금액을 표시한다

#### Scenario: 달러 사용 기록 실패 - 잘못된 입력
- **WHEN** 사용자가 음수의 금액을 입력한다
- **THEN** 시스템은 오류 메시지를 표시한다
- **AND** 데이터를 저장하지 않는다

## MODIFIED Requirements

### Requirement: Exchange Statistics Display
시스템 SHALL 현재까지의 환전 통계 정보를 표시할 수 있어야 한다.

#### Scenario: 통계 표시 성공
- **WHEN** 사용자가 통계 조회를 선택한다
- **THEN** 시스템은 총 환전 횟수를 표시한다
- **AND** 총 환전 달러 금액을 표시한다 (사용한 달러 포함하지 않음)
- **AND** 총 사용 달러 금액을 표시한다
- **AND** 현재 보유 달러를 표시한다 (총 환전 - 총 사용)
- **AND** 평균 환율(가중 평균)을 표시한다 (총 투자 원화 ÷ 총 환전 금액, 사용 금액 무관)
- **AND** 총 투자 원화 금액을 표시한다

#### Scenario: 통계 표시 - 데이터 없음
- **WHEN** 사용자가 통계 조회를 선택한다
- **AND** 저장된 환전 기록이 없다
- **THEN** 시스템은 "기록된 데이터가 없습니다" 메시지를 표시한다

### Requirement: Re-exchange Profit Calculator
시스템 SHALL 특정 환율에서 재환전 시 이익/손실을 계산할 수 있어야 한다.

#### Scenario: 이익 계산 성공
- **WHEN** 사용자가 목표 환율을 입력한다
- **THEN** 시스템은 현재 보유 달러(총 환전 - 총 사용)로 목표 환율 재환전 시 수익률을 계산한다
- **AND** 예상 수익(원화)을 표시한다
- **AND** 수익률(백분율)을 표시한다
- **AND** 이익/손실 여부를 표시한다
- **AND** 평균 환율은 변하지 않음을 표시한다

#### Scenario: 이익 계산 - 데이터 없음
- **WHEN** 사용자가 목표 환율을 입력한다
- **AND** 저장된 환전 기록이 없다
- **THEN** 시스템은 "먼저 환전 기록을 추가해주세요" 메시지를 표시한다

#### Scenario: 이익 계산 - 보유 달러 없음
- **WHEN** 사용자가 목표 환율을 입력한다
- **AND** 현재 보유 달러가 0이다
- **THEN** 시스템은 "현재 보유 달러가 없습니다" 메시지를 표시한다

### Requirement: Console User Interface
시스템 SHALL 사용자 친화적인 콘솔 인터페이스를 제공해야 한다.

#### Scenario: 메인 메뉴 표시
- **WHEN** 프로그램이 시작된다
- **THEN** 시스템은 다음 옵션들을 표시한다:
  - 1. 환율 기록 추가
  - 2. 달러 사용 기록 추가
  - 3. 전체 기록 보기
  - 4. 통계 보기
  - 5. 재환전 이익 계산
  - 6. 종료

#### Scenario: 메뉴 선택 및 실행
- **WHEN** 사용자가 메뉴 번호를 입력한다
- **THEN** 시스템은 해당 기능을 실행한다
- **AND** 실행 완료 후 메인 메뉴로 돌아간다

#### Scenario: 잘못된 메뉴 선택
- **WHEN** 사용자가 유효하지 않은 메뉴 번호를 입력한다
- **THEN** 시스템은 "잘못된 선택입니다" 메시지를 표시한다
- **AND** 메인 메뉴를 다시 표시한다

## ADDED Requirements

### Requirement: All Records Display
시스템 SHALL 환전 기록과 사용 기록을 구분하여 표시할 수 있어야 한다.

#### Scenario: 전체 기록 표시
- **WHEN** 사용자가 전체 기록 보기를 선택한다
- **THEN** 시스템은 모든 환전 기록과 사용 기록을 표시한다
- **AND** 환전 기록은 [IN] 태그로 표시한다
- **AND** 사용 기록은 [OUT] 태그와 용도를 표시한다
- **AND** 날짜순으로 정렬한다

#### Scenario: 기록 표시 - 데이터 없음
- **WHEN** 사용자가 전체 기록 보기를 선택한다
- **AND** 저장된 기록이 없다
- **THEN** 시스템은 "기록된 데이터가 없습니다" 메시지를 표시한다

## ADDED Requirements

### Requirement: Enhanced Data Persistence
시스템 SHALL 모든 환전 기록과 사용 기록을 CSV 파일에 영구 저장해야 한다.

#### Scenario: 데이터 자동 저장
- **WHEN** 사용자가 환전 기록 또는 사용 기록을 추가한다
- **THEN** 시스템은 즉시 CSV 파일에 데이터를 추가한다
- **AND** 파일이 없는 경우 새로 생성한다
- **AND** type 필드로 'exchange' 또는 'usage'를 구분한다

#### Scenario: 데이터 로드
- **WHEN** 프로그램이 시작된다
- **THEN** 시스템은 기존 CSV 파일에서 데이터를 로드한다
- **AND** type 필드를 기준으로 환전/사용 기록을 분류한다
- **AND** 파일이 없는 경우 빈 데이터로 시작한다
