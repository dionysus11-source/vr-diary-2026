## Requirements
### Requirement: 서버 기반 밸류리밸런싱 데이터 영속성 보존
시스템 SHALL 밸류리밸런싱 일지 및 투자 기록을 기존 브라우저 저장소가 아닌 백엔드 서버(DB)에 영구적으로 보존해야 한다(MUST).

#### Scenario: 교차 기기 및 브라우저 환경 동기화
- **WHEN** 사용자가 데스크톱에서 작업을 마친 후 모바일 등 다른 기기나 브라우저에서 밸류리밸런싱 메뉴에 접속한다
- **THEN** 서버로부터 최신 데이터를 불러와 동일한 밸류리밸런싱 투자 현황을 제공한다
- **AND** 브라우저 로컬 캐시는 오프라인 상태 보조용으로만 제한적으로 활용하며 서버 데이터베이스를 최우선 진실의 원천(Single Source of Truth)으로 사용한다

#### Scenario: 데이터 저장 및 API 통신 처리
- **WHEN** 사용자가 밸류리밸런싱 거래 기록을 추가 혹은 수정, 삭제한다
- **THEN** 즉각적으로 백엔드 서버 API를 호출하여 데이터베이스에 변경 상태를 반영한다
- **AND** 네트워크 에러나 응답 지연 발생 시 UI에 명확한 진행/실패 상태를 피드백(toast 알림 등)으로 제공한다

### Requirement: 왼쪽 사이드바 네비게이션
시스템 SHALL 왼쪽 사이드바에서 무한 매수법과 밸류 리밸런싱 전략을 선택할 수 있게 해야 한다(MUST).

#### Scenario: 사이드바 전환
- **WHEN** 사용자가 왼쪽 사이드바에서 "무한 매수법"을 선택한다
- **THEN** 메인 페이지(/)가 표시된다
- **AND** 기존 무한 매수법 페이지는 완전히 그대로 동작한다

#### Scenario: 밸류 리밸런싱 선택
- **WHEN** 사용자가 왼쪽 사이드바에서 "밸류 리밸런싱"을 선택한다
- **THEN** 밸류 리밸런싱 페이지(/value-rebalancing)가 표시된다
- **AND** URL이 /value-rebalancing로 변경된다

#### Scenario: 사이드바 상태 유지
- **WHEN** 사용자가 페이지를 새로고침한다
- **THEN** 현재 선택된 메뉴가 하이라이팅된다
- **AND** URL에 따라 활성 메뉴가 결정된다

### Requirement: 초기 포트폴리오 설정
사용자 SHALL 밸류 리밸런싱을 위한 초기 포트폴리오를 설정해야 한다(MUST).

#### Scenario: 초기 포트폴리오 입력
- **WHEN** 사용자가 밸류 리밸런싱을 처음 시작한다
- **THEN** 초기 현금(Pool), 초기 주식수, 평단가, 초기 날짜를 입력하라는 폼을 표시한다
- **AND** 총 투자 금액을 자동 계산하여 표시한다

#### Scenario: 초기 포트폴리오 저장
- **WHEN** 사용자가 초기 포트폴리오 정보를 입력하고 저장한다
- **THEN** 첫 V값을 계산한다 (V = E + pool/10)
- **AND** 벤치마크 주식수를 계산한다 (총 투자 금액 / 초기 종가)
- **AND** localStorage에 영구 저장한다

#### Scenario: 초기 포트폴리오 수정
- **WHEN** 사용자가 이미 설정된 포트폴리오를 수정하려고 한다
- **THEN** 현재 설정을 표시하고 수정을 허용한다
- **AND** 수정 후 관련 V값을 재계산한다

### Requirement: V값 계산 및 입력
시스템 SHALL 라오어 밸류 리밸런싱 공식으로 V값을 계산해야 한다(MUST).

#### Scenario: V2 계산 (기존 데이터 있음)
- **WHEN** 사용자가 현재 주식수, 현금(Pool), 종가를 입력한다
- **THEN** 평가금액 E를 계산한다 (종가 × 주식수)
- **AND** V2 = V1 + pool/10 + (E-V1)/6.32 공식으로 V2를 계산한다
- **AND** 소수점 둘째 자리까지 반올림하여 표시한다

#### Scenario: V값 계산 과정 표시
- **WHEN** 사용자가 V값 계산 결과를 본다
- **THEN** 계산 과정을 단계별로 표시한다:
  - 평가금액: E = 종가 × 주식수
  - V1 = 이전 V값
  - V2 = V1 + pool/10 + (E-V1)/6.32
- **AND** 각 단계의 숫자를 명확히 보여준다

#### Scenario: 가격 자동 조회
- **WHEN** 사용자가 날짜를 입력한다
- **THEN** 해당 날짜의 TQQQ 종가를 조회할 수 있는 옵션을 제공한다
- **AND** 수동 입력도 가능하게 한다

### Requirement: 리밸런싱 신호 시스템
시스템 SHALL V값 변화를 기반으로 매수/매도 신호를 제공해야 한다(MUST).

#### Scenario: BUY 신호
- **WHEN** V2가 V1 대비 15% 이상 상승한다
- **THEN** "BUY" 신호를 표시한다
- **AND** 녹색 색상으로 강조한다
- **AND** 변화율을 백분율로 표시한다

#### Scenario: SELL 신호
- **WHEN** V2가 V1 대비 15% 이상 하락한다
- **THEN** "SELL" 신호를 표시한다
- **AND** 빨간색 색상으로 강조한다
- **AND** 변화율을 백분율로 표시한다

#### Scenario: HOLD 신호
- **WHEN** V2가 V1 대비 ±15% 범위 내에 있다
- **THEN** "HOLD" 신호를 표시한다
- **AND** 회색 색상으로 표시한다
- **AND** 변화율을 백분율로 표시한다

#### Scenario: 신호 기록
- **WHEN** V값이 계산된다
- **THEN** 날짜, 종가, 주식수, Pool, E값, V값, 신호를 기록한다
- **AND** CSV 파일로 저장한다

### Requirement: 벤치마크 비교
시스템 SHALL 초기 전액 TQQQ 투자 시나리오와 실제 VR 전략 성과를 비교해야 한다(MUST).

#### Scenario: 벤치마크 계산
- **WHEN** 초기 포트폴리오가 설정된다
- **THEN** 벤치마크 주식수를 계산한다 (총 투자 금액 / 초기 종가)
- **AND** 벤치마크 가치를 계산한다 (벤치마크 주식수 × 현재 종가)

#### Scenario: 벤치마크 vs VR 비교
- **WHEN** 사용자가 현재 상태를 본다
- **THEN** VR 포트폴리오 가치 (E + Pool)를 표시한다
- **AND** 벤치마크 가치를 표시한다
- **AND** 차이를 금액과 백분율로 표시한다
- **AND** VR이 높으면 녹색, 낮으면 빨간색으로 표시한다

#### Scenario: 벤치마크 수익률 비교
- **WHEN** 사용자가 통계를 본다
- **THEN** VR 수익률과 벤치마크 수익률을 비교한다
- **AND** 초과 수익률을 표시한다

### Requirement: 차수 목록 및 상세 보기
시스템 SHALL 현재 진행중인 차수 목록을 보여주고 선택 시 상세 정보를 표시해야 한다(MUST).

#### Scenario: 진행중 차수 목록 표시
- **WHEN** 사용자가 밸류 리밸런싱 페이지를 연다
- **THEN** 현재 진행중인 차수 목록을 카드 형태로 표시한다
- **AND** 각 카드에 다음 정보를 표시한다:
  - 차수 번호 (예: "1차수", "2차수")
  - 시작일 ~ 종료일
  - 현재 V값
  - 현재 신호 (BUY/SELL/HOLD)
  - 평가금액
  - 현금(Pool)
  - 수익률
- **AND** 최신 차수가 상단에 표시된다

#### Scenario: 차수 선택 시 상세 보기
- **WHEN** 사용자가 차수 카드를 클릭한다
- **THEN** 해당 차수의 상세 정보가 펼쳐지거나 모달로 표시된다
- **AND** 상세 정보에는 다음이 포함된다:
  - 차수 기간 (시작일 ~ 종료일)
  - 초기 주식수/현금 vs 최종 주식수/현금
  - V값 변화 추이
  - 모든 리밸런싱 기록 (날짜별)
  - 매수/매도 거래 내역
  - 벤치마크와의 성과 비교
  - 차트 (V값 변화, 평가금액 추이)

#### Scenario: 완료된 차수 표시
- **WHEN** 차수가 종료되었다
- **THEN** "완료" 뱃지를 표시한다
- **AND** 진행중 차수와 구분하여 표시한다

### Requirement: 차수 데이터 관리
시스템 SHALL 2주 단위 차수로 매수/매도 기록을 관리해야 한다(MUST).

#### Scenario: 차수 자동 계산
- **WHEN** 새로운 리밸런싱 기록이 추가된다
- **THEN** 첫 기록일을 기준으로 2주 단위로 차수를 자동 할당한다
- **AND** 차수 ID를 생성한다 (예: "1차수", "2차수")
- **AND** 차수 기간(시작일, 종료일)을 계산한다

#### Scenario: 차수 상태 표시
- **WHEN** 사용자가 차수 목록을 본다
- **THEN** 각 차수의 현재 상태를 표시한다 (진행중/완료)
- **AND** 진행중 차수는 파란색, 완료된 차수는 회색으로 표시된다
- **AND** 차수 번호를 명확히 표시한다

#### Scenario: 차수별 매수/매도 기록
- **WHEN** 사용자가 차수 내에서 매수/매도를 한다
- **THEN** 해당 차수에 거래 기록을 추가한다
- **AND** 차수 시작/종료 시 주식수와 현금 변화를 추적한다
- **AND** 차수별 수익/손실을 계산한다
- **AND** 거래 내역을 차수 상세보기에서 표시한다

### Requirement: 시계열 차트
시스템 SHALL V값 변화와 성과를 시계열 차트로 시각화해야 한다(MUST).

#### Scenario: V값 변화 차트
- **WHEN** 사용자가 V값 차트를 본다
- **THEN** 날짜별 V값 변화를 선 그래프로 표시한다
- **AND** X축은 날짜, Y축은 V값이다
- **AND** BUY/SELL 신호 지점을 마커로 표시한다

#### Scenario: 평가금액 vs 벤치마크 차트
- **WHEN** 사용자가 성과 차트를 본다
- **THEN** VR 포트폴리오 가치와 벤치마크 가치를 비교 선 그래프로 표시한다
- **AND** VR은 실선, 벤치마크는 점선으로 표시한다
- **AND** 수익 구간은 녹색, 손실 구간은 빨간색으로 표시한다

#### Scenario: 차트 인터랙션
- **WHEN** 사용자가 차트의 데이터 포인트에 마우스를 올린다
- **THEN** 해당 날짜의 상세 정보를 tooltip으로 표시한다:
  - 날짜
  - 종가
  - V값
  - 평가금액
  - 벤치마크 가치
  - 신호
- **AND** tooltip을 클릭하면 해당 기록 상세를 볼 수 있다

### Requirement: 데이터 관리
시스템 SHALL 모든 리밸런싱 데이터를 영구 저장하고 관리해야 한다(MUST).

#### Scenario: 데이터 저장
- **WHEN** 사용자가 리밸런싱 기록을 추가/수정한다
- **THEN** 즉시 localStorage에 저장한다
- **AND** 저장 키는 "value_rebalancing_db"를 사용한다

#### Scenario: 데이터 로드
- **WHEN** 사용자가 페이지를 새로고침하거나 다시 방문한다
- **THEN** localStorage에서 모든 데이터를 로드한다
- **AND** 마지막 상태를 복원한다

#### Scenario: 데이터 초기화
- **WHEN** 사용자가 데이터 초기화를 요청한다
- **THEN** 확인 절차를 거친다 (이중 확인)
- **AND** 모든 리밸런싱 데이터를 삭제한다
- **AND** 초기 포트폴리오도 함께 삭제한다

#### Scenario: JSON 백업 생성
- **WHEN** 사용자가 백업/복원 버튼을 클릭한다
- **THEN** 현재 모든 리밸런싱 데이터를 JSON 파일로 다운로드한다
- **AND** 백업 파일에는 다음 정보가 포함된다:
  - 버전 정보
  - 내보낸 시각
  - 초기 포트폴리오 정보
  - 모든 리밸런싱 기록
  - 모든 차수 정보
  - 모든 거래 내역
- **AND** 파일명은 "value-rebalancing-backup-YYYY-MM-DD.json" 형식이다

#### Scenario: JSON 백업 복원
- **WHEN** 사용자가 백업 파일을 업로드한다
- **THEN** 백업 파일 형식을 검증한다
- **AND** 기존 데이터를 백업 파일의 데이터로 완전히 대체한다
- **AND** 복원 완료 메시지를 표시한다
- **AND** 페이지를 새로고침하여 복원된 데이터를 표시한다

#### Scenario: CSV Import/Export
- **WHEN** 사용자가 데이터를 백업하거나 복원하려고 한다
- **THEN** CSV 파일로 export할 수 있다
- **AND** CSV 파일에서 import할 수 있다
- **AND** CLI 프로그램과 호환되는 형식을 사용한다

### Requirement: 반응형 UI
밸류 리밸런싱 페이지 SHALL 다양한 화면 크기에서 최적으로 표시되어야 한다(MUST).

#### Scenario: 데스크톱 화면
- **WHEN** 사용자가 데스크톱(1024px 이상)에서 접속한다
- **THEN** 포트폴리오 상태, V값 폼, 차트가 2-3열 레이아웃으로 표시된다
- **AND** 차트는 넓게 표시된다

#### Scenario: 모바일 화면
- **WHEN** 사용자가 모바일(767px 이하)에서 접속한다
- **THEN** 모든 콘텐츠는 한 열로 표시된다
- **AND** 차트는 세로로 길게 표시된다
- **AND** 폼은 간결하게 표시된다

### Requirement: 직관적인 데이터 표시
시스템 SHALL 복잡한 V값 계산과 리밸런싱 개념을 직관적으로 표시해야 한다(MUST).

#### Scenario: 현재 포트폴리오 상태 카드
- **WHEN** 사용자가 밸류 리밸런싱 페이지를 연다
- **THEN** 상단에 현재 포트폴리오 상태를 요약하여 표시한다:
  - 현재 주식수
  - 현재 현금(Pool)
  - 평가금액(E)
  - 최신 V값
  - 최신 신호
  - 벤치마크 가치
  - VR vs 벤치마크 차이

#### Scenario: V값 계산 폼
- **WHEN** 사용자가 새로운 리밸런싱을 계산하려고 한다
- **THEN** 다음 필드를 제공한다:
  - 날짜 (기본값: 오늘)
  - TQQQ 종가 (자동 조회 또는 수동 입력)
  - 현재 주식수 (기본값: 이전 기록의 주식수)
  - 현재 현금(Pool) (기본값: 이전 기록의 현금)
- **AND** 계산 버튼을 제공한다

#### Scenario: 신호 색상 구분
- **WHEN** 신호가 표시된다
- **THEN** BUY는 녹색, SELL은 빨간색, HOLD는 회색으로 표시된다
- **AND** 배경색이나 테두리로도 구분한다

### Requirement: 도움말 및 문서
시스템 SHALL 밸류 리밸런싱 전략과 V값 공식에 대한 도움말을 제공해야 한다(MUST).

#### Scenario: V값 공식 설명
- **WHEN** 사용자가 V값 옆의 정보(ⓘ) 아이콘을 클릭한다
- **THEN** V값 공식을 설명하는 툴팁을 표시한다:
  - V2 = V1 + pool/10 + (E-V1)/6.32
  - 각 변수의 의미
  - 계산 예시

#### Scenario: 리밸런싱 신호 설명
- **WHEN** 사용자가 신호에 대한 설명을 원한다
- **THEN** BUY/SELL/HOLD 신호의 의미를 설명한다:
  - BUY: V값이 15% 이상 상승 → 매수 권고
  - SELL: V값이 15% 이상 하락 → 매도 권고
  - HOLD: ±15% 범위 내 → 대기

#### Scenario: 벤치마크 설명
- **WHEN** 사용자가 벤치마크에 대해 궁금해한다
- **THEN** 벤치마크 개념을 설명한다:
  - 초기 전액 TQQQ 투자 시나리오
  - VR 전략과의 차이점
  - 왜 비교가 중요한지

### Requirement: 현재 포트폴리오 수동 동기화
시스템 SHALL 사용자가 실제 계좌 상황과 다를 경우 주식수 및 현금(Pool)을 직접 수정하여 현재 포트폴리오 상태를 동기화할 수 있게 해야 한다(MUST).

#### Scenario: 현재 포트폴리오 정보 수정 시작
- **WHEN** 사용자가 현재 포트폴리오 상태 카드에서 "수정" 버튼을 클릭한다
- **THEN** 주식수와 현금(Pool)을 입력받을 수 있는 폼 모달을 표시한다
- **AND** 각 입력 필드는 현재(가장 최신) 포트폴리오 값으로 초기화되어 있어야 한다

#### Scenario: 포트폴리오 수정 완료
- **WHEN** 사용자가 수정된 주식수 및 현금을 입력하고 "저장" 버튼을 클릭한다
- **THEN** 서버 API를 호출하여 최신 기록(VRRecord)의 주식수와 현금을 변경된 값으로 업데이트한다
- **AND** 현재 기록의 eValue 및 totalValue 등 종속 데이터도 새 값에 맞게 재계산되어야 한다
- **AND** 업데이트 완료 시 화면에 반영된 최신 포트폴리오 상태가 표시된다
