# infinite-strategy Specification

## Purpose
무한매수법(Infinite Strategy) V2.2 및 V4.0 의 오늘자 최종 협의된 코어 로직(투자원금 기준, T값 디커플링, 스마트 분할 차수)과 사용자 인터페이스 기준을 정의한다.

## ADDED Requirements

### Requirement: 매입금액 (투자원금) 평단가 기준 표시
시스템은 사용자가 화면에서 확인하는 카드 및 모달의 '매입금액(구 투자원금)'을 이동평균법(평단가 × 잔존 수량) 기준으로 계산하여 표시해야 한다(MUST).

#### Scenario: 매입금액 표시
- **WHEN** 사용자가 무한매수법 진행 카드를 확인한다
- **THEN** '매입금액' 항목은 (잔존 주식 수 × 소수점 처리 전의 내부 정밀 평단가)로 산출된다
- **AND** 단순 누적 매수/매도액 합산 방식이 아닌, 현재 잔존 주식의 실질적인 평가 가치(Moving Average)를 반영하여 화면에 노출된다

### Requirement: T값(진행률) 순수 현금 흐름 기준 계산
시스템은 무한매수법의 기준이 되는 'T값(진행률)'을 화면에 보여지는 '매입금액'과 디커플링(분리)하여, 순수 현금 흐름(총 매수액 - 총 매도액) 기준으로 1회 시도액과 나누어 계산해야 한다(MUST).

#### Scenario: T값 계산 및 적용
- **WHEN** 자동 가이드 계산기가 백엔드에서 T값을 계산해낸다
- **THEN** 기준 투자 원금은 (총 누적 매수액 - 총 누적 매도액)으로 산출된다
- **AND** 이를 1회 시도액으로 나눈 값(예: 11.41)을 전체 알고리즘의 진행 지표로 삼는다

### Requirement: 진행도(Progress Bar) T값 동기화
시스템은 시드 소진율을 나타내는 '진행도 막대그래프'의 퍼센티지를 단순히 매입금액 기준이 아닌, T값 연산의 분자인 (총 매수액 - 총 매도액)을 바탕으로 '총 설정 시드 원금'과 나누어 표시해야 한다(MUST).

#### Scenario: 진행도 막대그래프 표시
- **WHEN** 무한매수법 중인 회차의 카드를 목록에서 본다
- **THEN** 진행도(%)는 (T값 산출용 순수 현금 흐름 원금 ÷ 총 시드 원금)으로 계산된다
- **AND** 이는 기술적으로 다른 앱의 (T값 ÷ 총 분할 차수) 와 완벽히 수학적으로 동일한 퍼센티지를 보장한다

### Requirement: 동적 분할 차수(Smart Split) 연동
시스템은 V2.2 및 V4.0 진행 시 기존 라오어 전략의 40분할 하드코딩 수치를 버리고, (총 시드 원금 ÷ 1회 시도액) 기반으로 동적 산출한 자체 분할 차수를 가이드라인 상태 전이에 활용해야 한다(MUST).

#### Scenario: V2.2 / V4.0 동적 가이드라인 전이
- **WHEN** 30분할 차수로 설정된 회차가 진행 중이다
- **THEN** '후반전(Second Half)' 상태 전이는 T값이 (30 ÷ 2)인 15 도달 시점부터 활성화된다
- **AND** '쿼터 손절모드(quarterStopLoss)' 상태 전이는 T값이 (30 - 0.9)인 29.1 도달 시 활성화된다
- **AND** 별(Star) 퍼센티지 점감률 또한 30분할의 기간에 비례해 더 빠르고 가파르게(10 ÷ 15) 동적으로 계산된다

### Requirement: 분할 차수 명시적 입력 기능
시스템은 새 회차 모달창을 띄울 때 '총 시드 원금'이나 '1회 시도액'과 더불어, 사용자가 직관적으로 '분할 차수'를 직접 지정할 수 있는 UI를 제공해야 한다(MUST).

#### Scenario: 새 회차 생성 시 자동 계산 기능
- **WHEN** 사용자가 새 회차 생성 모달에서 '총 시드 원금'에 20000을, '분할 차수'에 30을 입력한다
- **THEN** '1회 매수 시도액' 필드가 즉시 (20000 ÷ 30) 연산을 거쳐 666.67 달러로 자동 계산 및 반영된다
- **AND** 생성 버튼을 통해 등록된 회차 데이터는, V2.2 든 V4.0 이든 엔진 내부에서 즉시 30분할 차수에 맞물려 돌아가기 시작한다
