## Requirements
### Requirement: Exchange Rate Recording
시스템 SHALL 사용자가 환전 날짜, 환율, 금액을 기록할 수 있어야 한다.

#### Scenario: 환율 기록 성공
- **WHEN** 사용자가 환전 날짜, 환율(원/달러), 환전 금액(달러)을 입력한다
- **THEN** 시스템은 해당 정보를 CSV 파일에 저장한다
- **AND** 저장 성공 메시지를 표시한다

#### Scenario: 환율 기록 실패 - 잘못된 입력
- **WHEN** 사용자가 음수의 환율이나 금액을 입력한다
- **THEN** 시스템은 오류 메시지를 표시한다
- **AND** 데이터를 저장하지 않는다

### Requirement: Exchange Statistics Display
시스템 SHALL 현재까지의 환전 통계 정보를 표시할 수 있어야 한다.

#### Scenario: 통계 표시 성공
- **WHEN** 사용자가 통계 조회를 선택한다
- **THEN** 시스템은 총 환전 횟수를 표시한다
- **AND** 총 환전 달러 금액을 표시한다 (사용한 달러 포함하지 않음)
- **AND** 총 사용 달러 금액을 표시한다
- **AND** 현재 보유 달러를 표시한다 (총 환전 - 총 사용)
- **AND** 평균 환율(가중 평균)을 표시한다 (총 투자 원화 ÷ 총 환전 금액, 사용 금액 무관)
- **AND** 총 투자 원화 금액을 표시한다

#### Scenario: 통계 표시 - 데이터 없음
- **WHEN** 사용자가 통계 조회를 선택한다
- **AND** 저장된 환전 기록이 없다
- **THEN** 시스템은 "기록된 데이터가 없습니다" 메시지를 표시한다

### Requirement: Re-exchange Profit Calculator
시스템 SHALL 특정 환율에서 재환전 시 이익/손실을 정확하게 계산하고 색상으로 표시할 수 있어야 한다.

#### Scenario: 이익 계산 성공 (수정된 로직)
- **WHEN** 사용자가 목표 환율을 입력한다
- **THEN** 시스템은 현재 보유 달러의 원화 가치를 계산한다 (현재 보유 × 평균 환율)
- **AND** 목표 환율로 재환전 시 수익을 계산한다 (현재 보유 × 목표 환율)
- **AND** 이익/손실 = (재환전 후 가치 - 현재 보유 달러의 원화 가치)로 계산한다
- **AND** 이익인 경우 녹색으로 표시한다
- **AND** 손실인 경우 빨간색으로 표시한다
- **AND** 수익률을 백분율로 표시한다

#### Scenario: 이익 계산 예시
- **GIVEN** 현재 보유 $600, 평균 환율 ₩1,450/$, 목표 환율 ₩1,900/$
- **WHEN** 사용자가 목표 환율을 입력한다
- **THEN** 현재 보유 달러의 원화 가치: $600 × ₩1,450 = ₩870,000
- **AND** 재환전 후 가치: $600 × ₩1,900 = ₩1,140,000
- **AND** 이익: ₩270,000 (+31.03%)를 녹색으로 표시한다

#### Scenario: 손실 계산 예시
- **GIVEN** 현재 보유 $600, 평균 환율 ₩1,450/$, 목표 환율 ₩1,200/$
- **WHEN** 사용자가 목표 환율을 입력한다
- **THEN** 현재 보유 달러의 원화 가치: $600 × ₩1,450 = ₩870,000
- **AND** 재환전 후 가치: $600 × ₩1,200 = ₩720,000
- **AND** 손실: ₩150,000 (-17.24%)를 빨간색으로 표시한다

#### Scenario: 이익 계산 - 데이터 없음
- **WHEN** 사용자가 목표 환율을 입력한다
- **AND** 저장된 환전 기록이 없다
- **THEN** 시스템은 "먼저 환전 기록을 추가해주세요" 메시지를 표시한다

#### Scenario: 이익 계산 - 보유 달러 없음
- **WHEN** 사용자가 목표 환율을 입력한다
- **AND** 현재 보유 달러가 0이다
- **THEN** 시스템은 "현재 보유 달러가 없습니다" 메시지를 표시한다

### Requirement: Console User Interface
시스템 SHALL 사용자 친화적인 콘솔 인터페이스를 제공해야 한다.

#### Scenario: 메인 메뉴 표시
- **WHEN** 프로그램이 시작된다
- **THEN** 시스템은 다음 옵션들을 표시한다:
  - 1. 환율 기록 추가
  - 2. 달러 사용 기록 추가
  - 3. 전체 기록 보기
  - 4. 통계 보기
  - 5. 재환전 이익 계산
  - 6. 데이터 초기화
  - 7. 종료

### Requirement: Data Persistence
시스템 SHALL 모든 환전 기록을 CSV 파일에 영구 저장해야 한다.

#### Scenario: 데이터 자동 저장
- **WHEN** 사용자가 환전 기록을 추가한다
- **THEN** 시스템은 즉시 CSV 파일에 데이터를 추가한다
- **AND** 파일이 없는 경우 새로 생성한다

#### Scenario: 데이터 로드
- **WHEN** 프로그램이 시작된다
- **THEN** 시스템은 기존 CSV 파일에서 데이터를 로드한다
- **AND** 파일이 없는 경우 빈 데이터로 시작한다

### Requirement: Enhanced Data Persistence
시스템 SHALL 모든 환전 기록과 사용 기록을 CSV 파일에 영구 저장해야 한다.

#### Scenario: 데이터 자동 저장
- **WHEN** 사용자가 환전 기록 또는 사용 기록을 추가한다
- **THEN** 시스템은 즉시 CSV 파일에 데이터를 추가한다
- **AND** 파일이 없는 경우 새로 생성한다
- **AND** type 필드로 'exchange' 또는 'usage'를 구분한다

#### Scenario: 데이터 로드
- **WHEN** 프로그램이 시작된다
- **THEN** 시스템은 기존 CSV 파일에서 데이터를 로드한다
- **AND** type 필드를 기준으로 환전/사용 기록을 분류한다
- **AND** 파일이 없는 경우 빈 데이터로 시작한다

### Requirement: Color Output Support
시스템 SHALL Windows 10+ 및 Unix 계열 OS에서 ANSI 색상 코드를 지원해야 한다.

#### Scenario: 색상 출력 활성화
- **WHEN** 프로그램이 시작된다
- **THEN** 시스템은 ANSI 색상 코드를 지원하도록 설정한다
- **AND** Windows 10+에서는 콘솔 핸들을 설정하여 색상을 활성화한다
- **AND** 이익은 녹색(\033[92m), 손실은 빨간색(\033[91m)으로 표시한다

### Requirement: All Records Color Display
시스템 SHALL 환전 기록과 사용 기록을 색상으로 구분하여 표시할 수 있어야 한다.

#### Scenario: 전체 기록 표시 (색상 포함)
- **WHEN** 사용자가 전체 기록 보기를 선택한다
- **THEN** 시스템은 모든 환전 기록과 사용 기록을 표시한다
- **AND** 환전 기록은 녹색 [IN] 태그로 표시한다
- **AND** 사용 기록은 빨간색 [OUT] 태그와 용도를 표시한다
- **AND** 날짜순으로 정렬한다

#### Scenario: 기록 표시 - 데이터 없음
- **WHEN** 사용자가 전체 기록 보기를 선택한다
- **AND** 저장된 기록이 없다
- **THEN** 시스템은 "기록된 데이터가 없습니다" 메시지를 표시한다

### Requirement: Data Reset Functionality
시스템 SHALL 사용자가 모든 기록을 안전하게 초기화할 수 있어야 한다.

#### Scenario: 데이터 초기화 성공
- **WHEN** 사용자가 데이터 초기화를 선택한다
- **THEN** 시스템은 첫 번째 경고 메시지를 표시한다
- **AND** "모든 데이터가 영구적으로 삭제됩니다" 경고를 포함한다
- **AND** 현재 보유 달러와 총 기록 수를 표시한다
- **AND** 사용자에게 계속 진행할 것인지 묻는다
- **WHEN** 사용자가 확인한다 (예: 'yes' 입력)
- **THEN** 시스템은 두 번째 확인을 요청한다
- **AND** "정말로 모든 데이터를 삭제하시겠습니까?" 메시지를 표시한다
- **WHEN** 사용자가 다시 확인한다
- **THEN** 시스템은 CSV 파일을 삭제한다
- **AND** 메모리 상태를 초기화한다
- **AND** 성공 메시지를 표시한다

#### Scenario: 데이터 초기화 - 첫 번째 확인에서 취소
- **WHEN** 사용자가 데이터 초기화를 선택한다
- **THEN** 시스템은 첫 번째 경고 메시지를 표시한다
- **WHEN** 사용자가 취소한다 (예: 'no' 입력)
- **THEN** 시스템은 "초기화가 취소되었습니다" 메시지를 표시한다
- **AND** 메인 메뉴로 돌아간다
- **AND** 데이터는 유지된다

#### Scenario: 데이터 초기화 - 두 번째 확인에서 취소
- **WHEN** 사용자가 데이터 초기화를 선택한다
- **AND** 첫 번째 확인을 완료한다
- **AND** 두 번째 확인을 요청받는다
- **WHEN** 사용자가 취소한다
- **THEN** 시스템은 "초기화가 취소되었습니다" 메시지를 표시한다
- **AND** 메인 메뉴로 돌아간다
- **AND** 데이터는 유지된다

#### Scenario: 데이터 초기화 - 빈 데이터베이스
- **WHEN** 사용자가 데이터 초기화를 선택한다
- **AND** 저장된 기록이 없다
- **THEN** 시스템은 "삭제할 데이터가 없습니다" 메시지를 표시한다
- **AND** 메인 메뉴로 돌아간다
