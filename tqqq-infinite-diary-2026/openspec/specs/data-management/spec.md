## Requirements
### Requirement: 전체 데이터 백업 및 복원
시스템 SHALL 사용자가 무한매수 및 밸류리밸런싱 등 애플리케이션 내의 모든 비즈니스 대상 데이터를 한 번의 조작으로 일괄 백업 및 복원할 수 있는 기능을 제공해야 한다(MUST).

#### Scenario: 전체 데이터 백업 (Export)
- **WHEN** 사용자가 환경설정 혹은 데이터 관리 메뉴에서 '전체 데이터 백업' 버튼을 클릭한다
- **THEN** 서버(혹은 로컬 통합 저장소)에서 무한매수 일지와 밸류리밸런싱 등 모든 투자 기록을 취합하여 하나의 JSON 형태 백업 파일로 다운로드 제공한다
- **AND** 백업 파일명에는 백업 날짜와 시간 정보가 포함되어야 한다 (예: `infinite-diary-all-backup-20260319.json`)

#### Scenario: 전체 데이터 복원 (Import)
- **WHEN** 사용자가 이전에 백업해둔 '전체 데이터 백업' JSON 파일을 업로드하여 복원을 시도한다
- **THEN** 시스템은 유효한 백업 파일인지 포맷을 검증한다
- **AND** 검증 성공 시, 기존 데이터를 덮어쓸 것인지 명시적인 사용자 확인(Confirmation)을 받는다
- **AND** 사용자가 동의하면 무한매수와 밸류리밸런싱 관련 데이터를 일괄적으로 복원하고, 복원이 완료되면 새로운 데이터로 UI를 새로고침한다

#### Scenario: 데이터 복원 포맷 검증 실패
- **WHEN** 사용자가 손상되었거나 올바르지 않은 형식의 파일을 업로드하여 복원을 시도한다
- **THEN** 복원 프로세스를 즉각 중단하고 안전을 위해 기존 데이터를 보존한다
- **AND** 사용자에게 "지원하지 않는 파일 포맷이거나 데이터가 손상되었습니다"라는 오류 피드백을 노출한다

#### Scenario: 초기 데이터가 없을 때 복원 기능 접근
- **WHEN** 밸류리밸런싱 페이지에 진입했으나, 초기 포트폴리오 데이터가 아직 생성되지 않은 상태일 때
- **THEN** 기존 복원 흐름처럼 초기 포트폴리오 생성 폼과 함께 데이터 '백업/복원' 버튼을 화면 상단에 띄워서 복원을 먼저 수행할 수 있게 지원한다
