## Requirements
### Requirement: 매수/매도 기록 입력
시스템 SHALL 순차적인 매수/매도 기록 입력을 지원해야 한다(MUST).

#### Scenario: 순차적 입력 흐름
- **WHEN** 사용자가 새로운 거래를 입력한다
- **THEN** 다음 순서대로 입력 필드를 제공한다:
  1. 종목 선택 (TQQQ, SOXL 등 - 추후 추가 가능)
  2. 거래 유형 선택 (매수 또는 매도)
  3. 가격 입력
  4. 수량 입력
  5. 날짜 입력 (엔터치면 오늘 날짜 자동 입력, 한국시간 기준)

#### Scenario: 날짜 자동 입력
- **WHEN** 사용자가 날짜 입력 필드에서 엔터를 친다
- **THEN** 한국시간 기준 오늘 날짜를 자동으로 입력한다
- **AND** 한국시간 자정을 기준으로 날짜를 계산한다

#### Scenario: 날짜 시간대 처리
- **WHEN** 사용자가 날짜를 입력한다
- **THEN** 입력된 날짜는 한국시간(KST/UTC+9) 기준으로 해석된다
- **AND** 저장되는 날짜 형식은 YYYY-MM-DD(한국시간)이다
- **AND** 3월 5일로 입력된 데이터는 3월 5일 미국 시장 종가와 매칭된다
- **AND** 한국시간 날짜와 미국 시장 날짜를 1:1로 매칭한다

#### Scenario: 종목 관리
- **WHEN** 시스템이 초기화된다
- **THEN** 기본 종목으로 TQQQ, SOXL을 제공한다
- **AND** 관리자가 새로운 종목을 추가할 수 있다

### Requirement: 분할 매수 지원
시스템 SHALL 한 회차 내에서 여러 번의 매수(분할 매수)를 지원해야 한다(MUST).

#### Scenario: 분할 매수 입력
- **WHEN** 사용자가 진행중인 회차에 추가 매수를 입력한다
- **THEN** 해당 회차에 매수 기록을 추가한다
- **AND** 평균 매수 단가를 재계산한다: (기존 매수 총액 + 추가 매수액) / 총 수량
- **AND** 총 매수 수량을 누적한다

#### Scenario: 평균 단가 표시
- **WHEN** 사용자가 분할 매수가 있는 회차를 본다
- **THEN** 평균 매수 단가와 총 수량을 표시한다
- **AND** 각 매수별 상세 내역을 펼쳐서 볼 수 있다

### Requirement: 분할 매도 지원
시스템 SHALL 한 회차 내에서 여러 번의 매도(분할 매도)를 지원해야 한다(MUST).

#### Scenario: 분할 매도 입력
- **WHEN** 사용자가 진행중인 회차에 매도를 입력한다
- **THEN** 해당 회차에 매도 기록을 추가한다
- **AND** 보유 수량을 차감한다
- **AND** 실현 수익금을 계산한다
- **AND** 남은 보유 수량에 대한 미실현 수익을 계산한다

#### Scenario: 회차 자동 완료
- **WHEN** 회차의 보유 수량이 0이 된다
- **THEN** 회차 상태를 자동으로 "완료"로 변경한다
- **AND** 별도의 완료 처리가 필요 없다

#### Scenario: 보유 수량 확인
- **WHEN** 사용자가 매도를 입력하려고 한다
- **THEN** 현재 보유 수량을 표시한다
- **AND** 매도 수량이 보유 수량을 초과할 수 없다

### Requirement: 회차 탭 네비게이션
시스템 SHALL 대시보드 상단에서 회차 상태별 탭을 제공해야 한다(MUST).

#### Scenario: 탭으로 회차 필터링
- **WHEN** 사용자가 대시보드 상단의 탭을 선택한다
- **THEN** 선택한 탭에 해당하는 회차만 표시한다
- **AND** 탭 옵션은 "진행중", "완료", "전체" 순서로 정렬된다
- **AND** 초기 진입 시 기본 탭은 "진행중"으로 자동 선택된다
- **AND** 현재 선택된 탭이 시각적으로 강조된다

#### Scenario: 탭별 회차 카운트 표시
- **WHEN** 사용자가 대시보드를 본다
- **THEN** 각 탭에 해당 회차 수를 배지(badge)로 표시한다
- **AND** 예: "진행중 (3)", "완료 (12)"

### Requirement: 회차 카드 UI
시스템 SHALL 각 회차를 카드 형태로 표시해야 한다(MUST).

#### Scenario: 간결 모드 카드 표시
- **WHEN** 사용자가 카드 표시 모드를 "간결"로 선택한다
- **THEN** 각 카드에 다음 정보만 표시한다:
  - 회차 번호
  - 종목명 (TQQQ, SOXL 등)
  - 평균 매수 단가
  - 보유 수량 (총 매수 수량도 함께 표시)
  - 총 매수액
  - 총 수익률 (실현 + 미실현)
  - 총 수익금
  - 첫 매수 일자

#### Scenario: 상세 모드 카드 표시
- **WHEN** 사용자가 카드 표시 모드를 "상세"로 선택한다
- **THEN** 간결 모드 정보에 더해 다음을 추가로 표시한다:
  - 분할 매수 내역 (각 매수별 가격, 수량, 일자)
  - 분할 매도 내역 (각 매도별 가격, 수량, 일자, 있는 경우)
  - 실현 수익금 (매도가 있는 경우)
  - 미실현 수익금 (보유 수량이 있는 경우)
  - 수익률 미니 차트 (스파크라인)
  - 보유 기간

#### Scenario: 카드 표시 모드 전환
- **WHEN** 사용자가 카드 표시 모드 토글 버튼을 클릭한다
- **THEN** 간결/상세 모드 간에 전환된다
- **AND** 사용자의 선택이 localStorage에 저장된다

#### Scenario: 카드 색상 표시
- **WHEN** 회차 카드가 표시된다
- **THEN** 수익 회차는 초록색 테두리/배경을 가진다
- **AND** 손실 회차는 빨간색 테두리/배경을 가진다
- **AND** 진행중 회차는 파란색 테두리/배경을 가진다

#### Scenario: 카드 헤더 강조 표시
- **WHEN** 회차 카드가 표시된다
- **THEN** 종목명(예: TQQQ, SOXL)을 회차 번호보다 시각적으로 더 크고 굵은 폰트로 가장 눈에 띄게 배치한다
- **AND** 회차 번호는 종목명 옆에 뱃지 형태로 보조 정보로서 제공한다

### Requirement: 회차 상세 모달
시스템 SHALL 카드 클릭 시 회차 상세를 모달로 표시해야 한다(MUST).

#### Scenario: 모달 열기
- **WHEN** 사용자가 회차 카드를 클릭한다
- **THEN** 회차 상세 정보가 담긴 모달이 열린다
- **AND** 배경은 dimmed 처리된다
- **AND** ESC 키나 배경 클릭으로 모달을 닫을 수 있다

#### Scenario: 모달 내용 - 기본 정보 및 매수/매도 가이드
- **WHEN** 회차 상세 모달이 열린다
- **THEN** 기존 기본 정보(회차 번호, 종목명 등)와 함께, **무한매수법 V2.2 매수/매도 가이드 섹션**을 시각적으로 가장 잘 보이게 표시한다
- **AND** 상단에 현재 상태(전반전, 후반전, 쿼터손절) 뱃지와 함께 T값 및 Star값 카드를 배치한다
- **AND** 쿼터 손절모드에서 매도나 매수가 성립되어 후반전 복귀 조건을 충족하면, 즉각적으로 상태 뱃지가 '후반전'으로 다시 업데이트되어 현재 상태를 명확히 알 수 있게 표시한다

#### Scenario: 모달 내용 - 분할 매수 내역
- **WHEN** 회차에 분할 매수가 있다
- **THEN** 모달에 각 매수별 상세 내역을 표로 표시한다
- **AND** 테이블에는 매수 순서, 일자, 가격, 수량, 금액, 작업(수정/삭제 버튼)이 포함된다
- **AND** 총합계 행이 하단에 표시된다

#### Scenario: 모달 내용 - 분할 매도 내역
- **WHEN** 회차에 분할 매도가 있다
- **THEN** 모달에 각 매도별 상세 내역을 표로 표시한다
- **AND** 테이블에는 매도 순서, 일자, 가격, 수량, 금액, 작업(수정/삭제 버튼)이 포함된다
- **AND** 총합계 행이 하단에 표시된다
- **AND** 총 실현 수익금이 표시된다

#### Scenario: 모달 내용 - 시계열 수익률 차트
- **WHEN** 회차 상세 모달이 열린다
- **THEN** 해당 회차의 시간대별 수익률 변화를 차트로 표시한다
- **AND** X축은 시간(일자)을 나타낸다
- **AND** 주 Y축(왼쪽)은 종가($)를 나타낸다
- **AND** 보조 Y축(오른쪽)은 수익률을 나타낸다 (% 기호 제거)
- **AND** 매일 종가를 기준으로 수익률을 계산한다: (해당일 종가 - 평균 매수가) / 평균 매수가 × 100
- **AND** 평균 매수가 기준선을 수평 점선으로 명확히 표시한다
- **AND** 종가 선 그래프(파란색/검정색)와 수익률 영역 차트(초록색/빨간색)를 함께 표시한다
- **AND** 각 매수 시점에 "매수" 마커를 표시한다
- **AND** 각 매도 시점에 "매도" 마커를 표시한다
- **AND** 진행중 회차는 최신 종가로 차트를 자동 업데이트한다
- **AND** 완료된 회차는 최종 매도일까지의 추이를 보여준다
- **AND** 0% 수익률 기준선을 명확히 표시한다
- **AND** 수익 구간은 초록색, 손실 구간은 빨간색으로 표시한다
- **AND** API 로딩 중임을 표시한다
- **AND** 날짜는 한국시간 기준으로 표시된다
- **AND** 차트 하단 범례(Legend)는 표시하지 않는다

#### Scenario: 시계열 차트의 데이터 포인트 상세 보기
- **WHEN** 사용자가 시계열 차트의 데이터 포인트에 마우스를 올린다 (hover)
- **THEN** 해당 시점의 상세 정보를 tooltip으로 표시한다
- **AND** tooltip에는 다음 정보가 포함된다:
  - 일자 (한국시간 기준)
  - 종가 (예: $48.52)
  - 평균 매수가 (예: $45.30)
  - 수익률 (예: +7.11%)
  - 수익금 (예: +$3.22)
  - 해당 시점의 사건 (매수/매도/없음)
- **AND** 날짜는 한국시간 기준으로 표시된다

#### Scenario: 모달에서 편집 기능
- **WHEN** 사용자가 모달 하단의 편집 버튼을 클릭한다
- **THEN** 다음 기능을 제공한다:
  - **매수 추가**: 진행중 회차에 추가 매수 입력
  - **매도 추가**: 보유 수량이 있을 때 매도 입력
  - **거래 수정**: 개별 매수/매도 내역 수정 (가격, 수량, 날짜)
  - **거래 삭제**: 개별 매수/매도 내역 삭제
  - **회차 삭제**: 회차 전체 삭제 (확인 절차 포함)
  - **가격 새로고침**: 진행중 회차의 최신 종가 다시 가져오기

#### Scenario: 개별 거래 수정
- **WHEN** 사용자가 매수/매도 내역의 수정 버튼을 클릭한다
- **THEN** 수정 모달이 열린다
- **AND** 가격, 수량, 날짜를 수정할 수 있다
- **AND** 저장 후 관련 계산이 자동으로 업데이트된다
- **AND** 시계열 수익률 차트도 함께 업데이트된다

#### Scenario: 개별 거래 삭제
- **WHEN** 사용자가 매수/매도 내역의 삭제 버튼을 클릭한다
- **THEN** 삭제 확인 모달이 열린다
- **AND** 삭제할 거래의 정보를 보여준다
- **AND** 삭제 후 관련 계산이 자동으로 업데이트된다
- **AND** 매수 삭제 시 평균 매수가가 재계산된다
- **AND** 매도 삭제 시 실현 수익과 보유 수량이 재계산된다
- **AND** 시계열 수익률 차트도 함께 업데이트된다

### Requirement: 회차 자동 그룹핑
시스템 SHALL 매수 기록을 회차로 관리해야 한다(MUST).

#### Scenario: 신규 회차 생성
- **WHEN** 사용자가 새로운 매수를 입력하고 회차를 지정하지 않는다
- **THEN** 새로운 회차를 생성한다
- **AND** 회차 번호를 자동 부여한다 (마지막 회차 + 1)

#### Scenario: 기존 회차에 추가 매수
- **WHEN** 사용자가 기존 진행중 회차에 매수를 추가한다
- **THEN** 해당 회차에 매수 기록을 추가한다
- **AND** 평균 단가를 재계산한다

### Requirement: 무한 매수 기록 데이터 수집
시스템 SHALL 무한 매수 기록 데이터를 수집하고 서버 데이터베이스에 영구적으로 보존해야 한다(MUST).

#### Scenario: 데이터 수집 및 서버 저장 성공
- **WHEN** 유효한 무한 매수 기록 데이터가 제공된다
- **THEN** 데이터를 백엔드 API를 통해 서버 데이터베이스에 우선적으로 저장한다
- **AND** 각 회차의 다음 정보를 보존한다:
  - 회차 번호
  - 종목명 (TQQQ, SOXL 등)
  - 분할 매수 내역 (일자, 가격, 수량)
  - 분할 매도 내역 (일자, 가격, 수량, 있는 경우)
  - 평균 매수 단가
  - 총 매수 수량
  - 총 매도 수량
  - 보유 수량
  - 총 매수액
  - 총 매도액
  - 실현 수익률 및 수익금
  - 미실현 수익률 및 수익금
  - 총 수익률 및 수익금
  - 회차 상태 (진행중/완료)

#### Scenario: 서버 기반 이기종 환경 동기화
- **WHEN** 사용자가 PC 및 모바일 등 서로 다른 브라우저 환경에서 접속한다
- **THEN** 서버로부터 인증된 세션에 해당하는 최신 데이터를 불러와 동일한 화면을 제공한다
- **AND** 순수 로컬 캐시에만 의존하지 않고 항상 중앙 서버 데이터를 **Single Source of Truth** 로 취급한다

#### Scenario: 데이터 형식 검증
- **WHEN** 잘못된 형식의 데이터가 입력된다
- **THEN** 사용자에게 오류 메시지를 표시한다
- **AND** 데이터를 서버에 반영하지 않고 보호한다

### Requirement: 누적 수익금 추이 시각화
시스템 SHALL 전체 기간 동안의 누적 수익금 변화를 차트로 보여줘야 한다(MUST).

#### Scenario: 누적 수익금 표시
- **WHEN** 사용자가 누적 수익금 탭을 선택한다
- **THEN** 회차별 누적 자산 변화를 선 그래프로 표시한다
- **AND** 초기 투자금을 기준선(0%)으로 표시한다
- **AND** 현재 총 자산 가치를 명시한다

### Requirement: 요약 통계 표시
시스템 SHALL 무한 매수 기록의 핵심 통계 지표를 요약해서 보여줘야 한다(MUST).

#### Scenario: 기초 통계 표시
- **WHEN** 사용자가 대시보드를 본다
- **THEN** 다음 통계를 표시한다:
  - 총 회차 수
  - 완료된 회차 수
  - 평균 수익률
  - 최고 수익률 및 회차
  - 최저 수익률 및 회차
  - 승률 (수익 회차 / 전체 회차)

#### Scenario: 고급 통계 표시
- **WHEN** 사용자가 고급 분석을 요청한다
- **THEN** 추가적으로 다음을 표시한다:
  - 수익률 표준편차 (변동성)
  - 최대 손실폭 (MDD)
  - 평균 손실액
  - 평균 수익액
  - 손익비 (평균 수익 / 평균 손실)

### Requirement: MDD(Maximum Drawdown) 시각화
시스템 SHALL 최대 손실폭을 시각적으로 표시하여 리스크를 보여줘야 한다(MUST).

#### Scenario: MDD 차트 표시
- **WHEN** 사용자가 리스크 분석을 요청한다
- **THEN** 누적 수익금 그래프에 MDD 구간을 음영으로 표시한다
- **AND** 최대 손실폭 백분율을 명시한다
- **AND** MDD 발생 회차를 표시한다

### Requirement: 기간 필터링
사용자 SHALL 특정 기간의 데이터만 필터링해서 볼 수 있어야 한다(MUST).

#### Scenario: 회차 범위 필터
- **WHEN** 사용자가 시작 회차와 종료 회차를 선택한다
- **THEN** 해당 범위의 데이터만 모든 차트에 반영한다
- **AND** 필터링된 데이터의 통계를 다시 계산한다

#### Scenario: 최근 N회차 필터
- **WHEN** 사용자가 "최근 10회차"를 선택한다
- **THEN** 최근 10회치의 데이터와 통계를 표시한다

### Requirement: 연환산 수익률 계산
시스템 SHALL 회차 기간을 고려한 연간 환산 수익률을 계산해야 한다(MUST).

#### Scenario: 연환산 수익률 표시
- **WHEN** 각 회차의 평균 소요 일자가 계산 가능하다
- **THEN** 연환산 수익률을 계산하여 표시한다
- **AND** 계산식: (1 + 수익률)^(365/소요일수) - 1

### Requirement: 손실 복구 분석
시스템 SHALL 최대 손실 후 원금 회복까지 걸린 회차 수를 분석해야 한다(MUST).

#### Scenario: 복구 기간 표시
- **WHEN** MDD가 발생한 후 원금 회복이 있다
- **THEN** 손실 발생부터 회복까지 걸린 회차 수를 표시한다
- **AND** 평균 복구 기간을 계산한다

### Requirement: 데이터 내보내기
사용자 SHALL 분석 데이터를 CSV 형식으로 내보낼 수 있어야 한다(MUST).

#### Scenario: CSV 내보내기
- **WHEN** 사용자가 내보내기 버튼을 클릭한다
- **THEN** 현재 필터링된 데이터를 CSV 파일로 다운로드한다
- **AND** CSV에는 회차 번호, 일자, 수익률, 수익금이 포함된다

### Requirement: 직관적인 UI/UX
시스템 SHALL 직관적이고 사용하기 쉬운 사용자 인터페이스를 제공해야 한다(MUST).

#### Scenario: 명확한 시각적 계층 구조
- **WHEN** 사용자가任何 페이지에 접속한다
- **THEN** 중요한 정보가 상단에 배치된다
- **AND** 관련 있는 정보는 그룹화되어 표시된다
- **AND** 카드, 섹션 등으로 시각적으로 구분된다

#### Scenario: 수익/손실 직관적 표시
- **WHEN** 사용자가 수익률이나 수익금을 본다
- **THEN** 수익은 초록색(또는 파란색)으로 표시된다
- **AND** 손실은 빨간색으로 표시된다
- **AND** 양수는 "+" 접두사를 붙인다 (예: +12.5%)
- **AND** 음수는 "-" 접두사를 붙인다 (예: -8.3%)

#### Scenario: 대화형 피드백
- **WHEN** 사용자가 버튼이나 링크를 클릭한다
- **THEN** 로딩 상태를 시각적으로 표시한다 (스피너, skeleton UI)
- **AND** 성공/실패를 명확한 메시지로 알려준다 (toast notification)
- **AND** 오류 발생 시 해결 방법을 안내한다

#### Scenario: 간결한 입력 폼
- **WHEN** 사용자가 거래를 입력한다
- **THEN** 한 화면에 모든 필드가 보인다 (스크롤 최소화)
- **AND** 필수 필드는 명확히 표시된다
- **AND** 엔터키로 다음 필드로 이동할 수 있다
- **AND** 입력 완료 후 자동으로 다음 단계로 진행한다

### Requirement: 반응형 디자인
시스템 SHALL 다양한 화면 크기에서 최적으로 표시되어야 한다(MUST).

- **WHEN** 사용자가 모바일(767px 이하)에서 접속한다
- **THEN** 모든 콘텐츠는 한 열로 표시된다
- **AND** 차트는 가로 스크롤 가능하거나 세로로 길게 표시된다
- **AND** 통계 카드는 한 줄에 하나씩 표시된다
- **AND** 버튼은 엄지손가락으로 쉽게 tap할 수 있는 크기(최소 44px)다

### Requirement: 다크 모드 지원
시스템 SHALL 라이트/다크 모드를 지원해야 한다(MUST).

#### Scenario: 테마 전환

- **AND** 사용자의 선택이 localStorage에 저장된다
- **AND** 다음 방문 시 저장된 테마가 적용된다

#### Scenario: 시스템 테마 자동 감지

- **THEN** 운영체제의 테마 설정을 감지한다
- **AND** 다크 모드 설정 시 다크 테마를 적용한다

### Requirement: 데이터 로딩 및 에러 상태
시스템 SHALL 데이터 로딩 및 에러 상태를 명확히 표시해야 한다(MUST).

- **WHEN** 데이터를 불러오는 중이다
- **THEN** Skeleton UI나 스피너를 표시한다
- **AND** 로딩 중임을 명시하는 메시지를 보여준다

#### Scenario: 에러 상태 표시
- **WHEN** 데이터 로딩에 실패한다
- **THEN** 친절한 에러 메시지를 표시한다

- **AND** 첫 거래를 추가하는 버튼을 제공한다
- **AND** 빈 상태를 나타내는 일러스트레이션을 표시한다

### Requirement: 접근성
시스템 SHALL 웹 접근성 표준을 준수해야 한다(MUST).

- **THEN** Tab 키로 모든 인터랙티브 요소에 접근할 수 있다
- **AND** Enter/Space 키로 버튼을 활성화할 수 있다

- **THEN** 모든 이미지에 alt 텍스트가 제공된다

### Requirement: 주식 종가 데이터 가져오기
시스템 SHALL 외부 API를 통해 주식 종가 데이터를 자동으로 가져와야 한다(MUST).

#### Scenario: 일별 종가 데이터 가져오기
- **WHEN** 회차 상세를 조회한다
- **THEN** 해당 회차 기간(첫 매수일부터 현재 또는 최종 매도일)의 일별 종가 데이터를 API로 가져온다
- **AND** 종가 데이터에는 날짜와 종가(Close price)가 포함된다
- **AND** API 요청 실패 시 재시도하거나 적절한 에러 메시지를 표시한다

#### Scenario: 종가 데이터 날짜 매칭
- **WHEN** 한국시간 3월 5일에 입력된 매수/매도 데이터의 종가를 가져온다
- **THEN** 3월 5일 미국 시장 종가 데이터와 매칭한다
- **AND** Yahoo Finance API의 3월 5일 종가 데이터를 사용한다
- **AND** 한국시간 날짜와 미국 시장 날짜를 1:1로 매칭한다

#### Scenario: 종가 데이터 캐싱
- **WHEN** 이미 가져온 날짜 범위의 종가 데이터를 다시 요청한다
- **THEN** 캐시된 데이터를 사용하여 불필요한 API 호출을 방지한다
- **AND** 캐시는 localStorage에 저장된다
- **AND** 캐시는 매일 한국시간 자정에 만료된다
- **AND** 진행중 회차의 경우 최신 데이터를 자동으로 갱신한다

#### Scenario: 진행중 회차 실시간 가격

- **THEN** 최신 종가 데이터를 자동으로 가져와 시계열 차트를 업데이트한다
- **AND** 사용자가 "새로고침" 버튼을 클릭하면 최신 가격을 다시 가져온다

#### Scenario: 다중 종목 지원

- **THEN** 각 종목 심볼에 맞는 종가 데이터를 반환한다
- **AND** 향후 새로운 종목 추가 시 API 설정만으로 확장 가능하다

### Requirement: 무한매수법 V2.2 가이드 제공
시스템 SHALL 회차의 현재 진행 상태(누적 매수 금액, 1회 매수 시도액 등)를 기반으로 무한매수법 V2.2 매수/매도 가이드를 제공해야 한다(MUST).

#### Scenario: 전반전 매수/매도 가이드 표시
- **WHEN** 현재 회차의 T값(누적 매수액 / 1회 매수 시도액)이 20 미만인 전반전일 때
- **THEN** 1회 매수액의 절반은 평단가 LOC로, 나머지 절반은 평단가 대비 별(Star)퍼센트 LOC로 매수하도록 가이드를 제공한다
- **AND** 폭락장 대비 추가 매수 항목들을 표시한다
- **AND** 매도 가이드로 전체 수량의 1/4은 평단가 대비 별퍼센트 LOC, 나머지 3/4은 지정가 10%에 매도하도록 가이드한다

#### Scenario: 후반전 매수/매도 가이드 표시
- **WHEN** 현재 회차의 T값이 20 이상 39.1 미만인 후반전일 때
- **THEN** 매수 시도액 전체를 평단가 대비 별퍼센트 LOC로 매수하도록 가이드한다
- **AND** 매도 가이드로 전체 수량의 1/4은 평단가 대비 별퍼센트 LOC, 나머지 3/4은 지정가 10%에 매도하도록 가이드한다

#### Scenario: 쿼터 손절모드 가이드 표시
- **WHEN** 1회 매수 시도 금액이 부족해지거나 T값이 39.1 이상일 때 (원금 소진)
- **THEN** 현재 진행 상태가 '쿼터 손절모드'임을 알리고 1차적으로 전체 보유 수량의 1/4를 시장가(MOC)로 당장 매도하라는 가이드를 표시한다
- **AND** 남은 자금을 10회로 분할하여 재진입하는 쿼터손절 매수(-10% LOC 매수 걸기) 및 매도(-10% LOC에 1/4 분할 매도, 나머지 +10% 지정가) 안내를 제공한다
- **AND** 후반전 정상 복귀 조건(+10% 지정가 체결 또는 매도 1회 성립 시 복귀)을 추가로 명시한다

### Requirement: T값 및 Star값 계산 내역 제공
시스템 SHALL 가이드에 사용된 핵심 지표인 T값과 Star값의 도출 과정을 사용자에게 직관적으로 제공해야 한다(MUST).

#### Scenario: 정보 아이콘 클릭 시 계산 과정 표시
- **WHEN** 사용자가 T값 또는 Star값 옆의 정보(ⓘ) 아이콘을 클릭하거나 마우스를 올린다(hover)
- **THEN** 현재 누적 매수액 데이터와 1회 시도액을 바탕으로 어떻게 해당 값이 도출되었는지 숫자 계산 과정을 표시한다
- **AND** 예시로 T값의 경우 `(누적매수액) / (1회 시도액) = 결과값 (소수점 둘째자리 올림)` 구조로 보여준다
- **AND** Star값의 경우 `(10 - T/2)%` 수식에 현재 T값을 직접 대입한 연산 처리 과정을 표시하여 검증할 수 있게 지원한다
